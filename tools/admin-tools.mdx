The admin tools expose bucket and account metadata by calling the Cloudflare API on behalf of the MCP client. All ten tools are strictly read-only — they surface configuration and metrics but never create, update, or delete any account resource. This constraint is intentional: read-only access limits the blast radius of a compromised session to information disclosure only. The tools are **disabled by default** and require a Cloudflare API token with a narrowly scoped R2 permission.

See [Tools Overview](/tools/overview) for the full tool surface and [Deployment Configuration](/deployment/configuration) for API token setup details.

## Enabling Admin Tools

Add the following to the `vars` block of your `wrangler.jsonc`:

```jsonc
"vars": {
  "ENABLE_ACCOUNT_TOOLS": "true",
  "CLOUDFLARE_ACCOUNT_ID": "your-account-id",
  "R2_BUCKET_NAME": "your-bucket-name"
}
```

Then store the API token as a Wrangler secret:

```sh
npx wrangler secret put CLOUDFLARE_API_TOKEN -c wrangler.jsonc
```

<Note>
Use the narrowest readable R2 scope available when creating the API token. A token scoped to **R2 Read** (or equivalent) is sufficient for all ten admin tools. Never use a global or zone-level token — if the token is ever compromised, the damage is contained to read-only R2 data.
</Note>

## All Admin Tools

Every admin tool carries `readOnlyHint: true` in its MCP annotations, signalling to clients that no state will be mutated.

| Tool | Description |
|---|---|
| `r2_bucket_list` | List R2 buckets in the account |
| `r2_bucket_get` | Get metadata for one bucket |
| `r2_cors_get` | Get CORS rules for one bucket |
| `r2_lifecycle_get` | Get lifecycle rules for one bucket |
| `r2_domain_custom_list` | List custom domains for one bucket |
| `r2_domain_custom_get` | Get custom domain settings for one bucket and domain |
| `r2_domain_managed_get` | Get r2.dev managed domain settings for one bucket |
| `r2_notifications_list` | List event notification rules for one bucket |
| `r2_notifications_get` | Get one event notification queue configuration |
| `r2_metrics_get` | Get account-level R2 storage metrics |

## Tool Reference

### r2_bucket_list

Lists all R2 buckets in the configured Cloudflare account. Supports pagination and server-side filtering by name.

<ParamField body="cursor" type="string">
  Pagination cursor returned by a previous `r2_bucket_list` response. Omit for the first page.
</ParamField>

<ParamField body="direction" type="string">
  Sort direction for the bucket list. Accepted values: `"asc"` or `"desc"`.
</ParamField>

<ParamField body="nameContains" type="string">
  Filter results to buckets whose names contain this substring.
</ParamField>

<ParamField body="perPage" type="integer">
  Maximum number of buckets to return per page. Must be between `1` and `1000`.
</ParamField>

**Example call:**

```json
{
  "nameContains": "prod",
  "direction": "asc",
  "perPage": 50
}
```

---

### r2_bucket_get

Returns metadata for a single R2 bucket, including creation time and location hint.

<ParamField body="bucketName" type="string">
  Name of the bucket to inspect. Defaults to the value of `R2_BUCKET_NAME` when omitted.
</ParamField>

**Example call:**

```json
{
  "bucketName": "my-assets"
}
```

---

### r2_cors_get

Returns the CORS rules currently configured on a bucket. Useful for verifying that browser-origin restrictions are set correctly before enabling presigned URL flows.

<ParamField body="bucketName" type="string">
  Name of the target bucket. Defaults to `R2_BUCKET_NAME` when omitted.
</ParamField>

**Example call:**

```json
{
  "bucketName": "my-assets"
}
```

---

### r2_lifecycle_get

Returns the object lifecycle rules for a bucket, including expiration and transition policies.

<ParamField body="bucketName" type="string">
  Name of the target bucket. Defaults to `R2_BUCKET_NAME` when omitted.
</ParamField>

---

### r2_domain_custom_list

Lists all custom domains attached to a bucket.

<ParamField body="bucketName" type="string">
  Name of the target bucket. Defaults to `R2_BUCKET_NAME` when omitted.
</ParamField>

---

### r2_domain_custom_get

Returns the settings for one specific custom domain attached to a bucket.

<ParamField body="bucketName" type="string">
  Name of the target bucket. Defaults to `R2_BUCKET_NAME` when omitted.
</ParamField>

<ParamField body="domain" type="string" required>
  The custom domain name to look up (e.g., `"assets.example.com"`).
</ParamField>

**Example call:**

```json
{
  "bucketName": "my-assets",
  "domain": "assets.example.com"
}
```

---

### r2_domain_managed_get

Returns the r2.dev managed subdomain settings for a bucket, including whether public access is enabled.

<ParamField body="bucketName" type="string">
  Name of the target bucket. Defaults to `R2_BUCKET_NAME` when omitted.
</ParamField>

---

### r2_notifications_list

Lists all event notification rules configured on a bucket. Each rule maps bucket events (such as object creation or deletion) to a Cloudflare Queue.

<ParamField body="bucketName" type="string">
  Name of the target bucket. Defaults to `R2_BUCKET_NAME` when omitted.
</ParamField>

---

### r2_notifications_get

Returns the full configuration for one event notification queue associated with a bucket.

<ParamField body="bucketName" type="string">
  Name of the target bucket. Defaults to `R2_BUCKET_NAME` when omitted.
</ParamField>

<ParamField body="queueId" type="string" required>
  The Queue ID of the notification configuration to retrieve.
</ParamField>

**Example call:**

```json
{
  "bucketName": "my-assets",
  "queueId": "abc123def456"
}
```

---

This tool takes no parameters.

## Intentional Omissions

<Note>
All admin tools are annotated with `readOnlyHint: true`. No tool in this surface will ever mutate account state.
</Note>

The following mutating account operations are deliberately **not implemented**:

| Category | Omitted operations |
|---|---|
| Buckets | Create bucket, delete bucket |
| CORS | Put CORS rules, delete CORS rules |
| Lifecycle | Put lifecycle rules, delete lifecycle rules |
| Custom domains | Attach domain, update domain, remove domain |
| Managed domain | Enable/disable r2.dev public access |
| Notifications | Create notification rule, update notification rule, delete notification rule |

These omissions are by design. A read-only tool surface means:

- A compromised MCP session or API token cannot alter bucket configuration, destroy data pipelines, or expose buckets publicly.
- The blast radius of any incident is limited to information disclosure of account metadata.
- Operators can safely enable admin tools in production without gating them behind additional change-control workflows.

If you need to perform any of the omitted operations, use the [Cloudflare Dashboard](https://dash.cloudflare.com) or the Cloudflare API directly outside of the MCP session.
